# GSI Dashboard — Session Learnings Log
# ════════════════════════════════════════════════════════════════════════
#
# PURPOSE: Tracks what each Claude session learned fresh, what context
# was stale or wrong, and any near-miss confusions. Enables detection of:
#   - Hallucinations (Claude invented something not in the codebase)
#   - Stale information (CLAUDE.md / context files lagging reality)
#   - Confusion patterns (Claude misread a file or misapplied a rule)
#   - Genuine new learnings (patterns discovered during the session)
#
# RULES (append-only — same discipline as GSI_AUDIT_TRAIL.md):
#   1. One record per session. Written by Claude at session close.
#   2. Records are never edited. Use type: CORRECTION to amend a prior record.
#   3. Every stale-info finding must note WHICH file was wrong and the fix.
#   4. Hallucinations get flagged even if minor — pattern detection requires
#      complete data.
#   5. Updated by Claude via /log-learnings command (Phase 3 close step).
#
# RECORD TYPES:
#   LEARNING   — new understanding gained that isn't in any doc
#   STALE      — a context file contained outdated information
#   CONFUSION  — Claude misread or misapplied something (near-miss)
#   HALLUCINATION — Claude generated a fact not present in the codebase
#   CORRECTION — amends a prior record (reference by session ID)
#
# FORMAT:
#   ## RECORD-NNN | YYYY-MM-DD | session_NNN | vX.XX | TYPE
#   **Finding:** [what was learned / what was wrong]
#   **Source:** [which file or interaction revealed this]
#   **Impact:** LOW | MEDIUM | HIGH
#   **Fix applied:** [what was corrected, or "none — documentation only"]

## RECORD-012 | 2026-04-07 | session_019 | v5.36 | LEARNING
**Finding:** LiteLLM proxy environment variables (`ANTHROPIC_BASE_URL`, `ANTHROPIC_AUTH_TOKEN`) must be exported **before** launching `claude` — they cannot be set after launch. Each Bash tool call runs in an isolated subshell that does not propagate changes back to the parent process. Even if `export VAR=val` succeeds in a Bash tool call, the already-running Claude Code binary inherits its env at fork time and is unaffected. The "set vars after `/new-session`" approach in the original proxy startup guide was therefore fundamentally broken at the OS level.
**Source:** Attempted `export ANTHROPIC_BASE_URL=...` via Bash tool; subsequent `curl` confirmed env vars visible in subshell but `python3 -c "import os; print(os.environ.get('ANTHROPIC_BASE_URL'))"` confirmed the Claude process itself still saw `None`.
**Impact:** MEDIUM — delayed proxy work; wasted tokens diagnosing; proxy items D-02/OPEN-006/EQA-41 run under subscription instead. Documented as PROXY-08 in backlog.
**Fix applied:** CLAUDE.md proxy startup guide and `sprint_planner.py` YELLOW warning both corrected to show the two-launch sequence. ADR-024 recorded.
