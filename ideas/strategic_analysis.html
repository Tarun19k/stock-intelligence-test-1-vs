THE CROWN JEWEL · SYNTHESIS PIPELINE

Event → Impact → Signal — 5 Stages

The LLM is the scalpel, not the conveyor belt. Expected survival rate: 10–20% of raw events reach Stage 3 (expensive). All knobs live in admin_config.

STAGE 1
Dedupe & Cluster
pgvector cosine similarity clusters duplicate news within 48h. Only cluster representatives proceed.
~$0 · local model
STAGE 2
Relevance Filter
Entity/graph match against dependency edges + small-model YES/NO check. Hard daily call cap. Expected drop: 80–90%.
Low cost · capped
STAGE 3
LLM Synthesis
Full Anthropic call on survivors only. JSON output: affected instruments, direction, magnitude, confidence, rationale, proposed graph edges.
High cost · budget guarded
STAGE 4
Materiality Gate
score = confidence × magnitude. Only score ≥ synthesis.materiality_min triggers a signal update. Sub-threshold results stored but not surfaced.
~$0 · pure logic
STAGE 5
Signal Update + Audit
UPSERT signals. INSERT signal_audit (append-only). Decay job runs nightly — stale news doesn't pin signals forever. Admin approves proposed graph edges.
~$0 · DB write
Worked example (Spec §9.3 acceptance test): "Cabinet approves mining royalty changes" → S1: 6 outlet copies → 1 representative. S2: graph match hits policy_domain:mining edges → miners, metal consumers, capital-goods suppliers. S3: Coal India ↑ (0.7×0.8=0.56), Hindalco ↓ (0.5×0.6=0.30), unheld mining-services firm flagged as alpha. S4: all pass 0.35 threshold. S5: 3 signal updates + audit rows; alpha candidate forwarded to advise engine.

FRAMEWORK 5 · UNBIASED OBSERVER

What We're Not Seeing

Stoic Phantasia — what an unbiased witness reports that the decision-maker is not measuring.

1

LLM Cost Spiral — No Hard Architectural Cap

The daily_budget_usd in admin_config is a manual knob, not a hard guard enforced in code. As the held universe grows from 1 ticker (G1) to full portfolio (G2+), synthesis event volume scales. There's no circuit breaker in synthesis/llm.py — only admin config that a busy session might not check.

Instrument needed: real-time synthesis cost dashboard + hard cap in llm.py pre-call check
2

"DONE (plan)" ≠ "DONE (verified)"

18 of 29 gaps are marked with a status like "DONE (schema/plan)" or "DONE (note)" — meaning the solution was designed on paper but not implemented or tested. The gap register doesn't distinguish between a verified, running fix and a documented intention. This creates a false-confidence floor going into Phase 0.

Instrument needed: a verification column in GAP_REGISTER with ✓ tested / planned / open
3

Mobile-First Is Stated But Not Gated Until Phase 6

GAP-28 says "Tarun lives on mobile — mobile is the default surface." But the Playwright default viewport is noted there, and actual mobile-responsive design is Phase 6. That means G1 through G5 UIs are built without mobile validation — 5 phases of potentially desktop-first decisions that become retrofit work at G6.

Instrument needed: mobile viewport in Playwright from Phase 1.5 onward — not Phase 6.2

FRAMEWORK 6 · STRATEGIC VERDICT

Posture & Leverage

STRATEGIC POSTURE
CONSOLIDATE
The plan has superb structural enforcement (Pillar 6 = 5/5) but two pillars score 2/5 — Allies (data sourcing) and Market (user definition). Extending scope before shoring these up risks building toward an undefined objective on a legally fragile foundation.
01

EODHD trial + adapter in Phase 0 backlog

Commit to the licensed replacement NOW, not at G7. Build the EODHD adapter in parallel with Phase 1. The migration cost compounds with every phase built on yfinance alone. A $20/mo subscription resolves GAP-15 structurally.

02

Risk profile + 2 user personas before G5

The rebalancing engine (cvxpy, Phase 5) optimizes a portfolio — but toward what? Without Tarun's goals, time horizon, and risk tolerance defined, the G5 output is generic optimization, not personal investment guidance. This is the spec's own stated north star.

03

Hard LLM budget cap in synthesis/llm.py — not just admin_config

Wire the daily budget as a pre-call assertion in code, not a config value that requires manual monitoring. Cost spirals are not self-announcing — they show up in the bill, not the logs. GAP-21's Batches API savings should be Phase 4 architecture, not Phase 4.6.

⛔ ONE THING TO STOP DOING IMMEDIATELY

Treating "DONE (plan)" gaps as resolved. 18 of 29 gaps in the register are plan-level closures — designed on paper but not tested. Before Phase 0 begins, add a verification column to GAP_REGISTER.md. A gap that hasn't been tested in a running system is still open. False-completed gaps are the most dangerous kind.
