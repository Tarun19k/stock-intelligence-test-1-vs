6 Mandatory Accuracy Guards
1
Regime-segmented accuracyWeights adjust within RISK_ON/OFF/STAGFLATION/DEFLATION, never across regimes. Prevents cross-cycle weight contamination.
2
Counter-cyclical weightingAccuracy streaks (N ≥ 5 consecutive correct) trigger skepticism via STREAK_DISCOUNT_FACTOR = 0.7. Prevents reflexivity at market peak.
3
Fundamentals weight floorROIC + FCF + promoter pledge combined weight never drops below FUNDAMENTAL_WEIGHT_FLOOR = 0.30 regardless of ledger score.
4
24-segment accuracy ledgerLynch classification (6) × Dalio regime (4) = 24 independent segments. Fast Grower weights never bleed into Cyclical analysis.
5
Magnitude + peak-return loggingSignal quality = hit rate × magnitude factor. Prevents optimising for frequency instead of return capture.
6
Quarterly human review gateWeight proposals generated automatically, never auto-applied. Tarun approves or rejects every quarter. Proposal banner in Streamlit UI.

EXIT Trigger Conditions — path optimizer

All four conditions log to trade_outcomes.exit_trigger (E1–E4). E2 requires 3 consecutive BEAR emits — not a single flip.

E1
Position drift outside Kelly bandcurrent_value > max_position × 1.1 or < min_position × 0.9 → Rebalance
E2
Signal direction reversal — 3× confirmedBULL → BEAR on 3 consecutive emits for same instrument → Exit (reversal confirmed, not noise)
E3
Insufficient edgemagnitude_target drops below 3% → signal no longer asymmetric enough to hold
E4
Sector cap breachHolding or adding would push sector allocation above 35% for the bucket → Exit or block
f* = ( p − q ÷ b ) × ¼
Quarter-Kelly position sizing  ·  p = confidence / 100  ·  q = 1 − p  ·  b = magnitude_target (floor 5%)
Position clamped: max( ₹7,250 , min( ₹72,500 , portfolio_value × f* ) )
